BotSpot delivery_mode fixture

Purpose: Verify the delivery_mode dispatch logic end-to-end against a running test server.
  1. Start the test server: cd testing && python test-server.py
  2. In the WP admin, set the locus API URL to http://localhost:5000.
  3. Use the scenarios below to set delivery_mode on a test page via the dashboard, then view the page and inspect source.
Expected results by mode:
  • disabled: No <script type="application/ld+json"> tag in <head>. No [data-bsa-appendix] in body.
  • jsonld_only: One <script type="application/ld+json"> tag present. No [data-bsa-appendix] in body.
  • full: Both JSON-LD tag and [data-bsa-appendix] present.
  • Missing field (older core): Treated as full.

Scenario 1: delivery_mode = disabled

Configure a test page with injection_overrides = {"delivery_mode": "disabled"} in the dashboard. Load the page and verify:

  • No <script type="application/ld+json"> in page source.
  • No data-bsa-appendix element anywhere in the rendered HTML.
  • No BotSpot HTML comment markers.

Check via: view-source:<page-url> and search for "BotSpot" and "bsa-appendix".

Scenario 2: delivery_mode = jsonld_only

Configure a test page with injection_overrides = {"delivery_mode": "jsonld_only"}. Load the page and verify:

  • One <script type="application/ld+json"> tag with BotSpot attribution in <head>.
  • No data-bsa-appendix element in body.
  • The BotSpot JSON-LD comment markers are present (<!-- BotSpot JSON-LD -->).

Scenario 3: delivery_mode = full (or unset)

Configure a test page with injection_overrides = {"delivery_mode": "full"} or remove delivery_mode entirely (older core response). Verify:

  • JSON-LD <script> tag present in <head>.
  • [data-bsa-appendix] element present in body with rendered appendix HTML.

Scenario 4: Missing delivery_mode field (backward compat)

Point the plugin at an older core instance that returns a /render response without a delivery_mode field. Verify the plugin treats this as full and injects both JSON-LD and appendix HTML.

Simulate by returning a response body like: {"html": "<p>test</p>", "jsonld": {...}, "content_hash": "abc"} (no delivery_mode key).
